# Trouble Shooting

본 문서는 개발 및 운영 과정에서 경험한 문제 상황과 해결 과정을 정리한 내용입니다.

---

## 1. Interface Server 응답 데이터 처리 문제

### 📌 문제 상황

Interface Server와 연동하는 과정에서 일부 API 응답 값이 정상적으로 화면에 표시되지 않는 문제가 발생했습니다.

### 🔍 원인

Interface Server에서 전달되는 응답 데이터의 구조가 일정하지 않거나,
일부 필드 값이 null로 전달되는 경우가 있었습니다.

초기 구현에서는 이러한 예외 케이스를 충분히 고려하지 못해
데이터 처리 과정에서 오류가 발생했습니다.

### 🛠 해결 방법

* 응답 데이터에 대한 null 체크 및 유효성 검증 로직 추가
* 예상하지 못한 값에 대한 기본 처리 로직 구현
* 서비스 계층에서 응답 파싱 로직을 공통화하여 안정성 확보

### 📈 결과

* 데이터 처리 안정성 향상
* 운영 화면에서 발생하던 오류 감소

### 💡 배운 점

외부 시스템과 연동할 때는 항상 예상치 못한 데이터가 들어올 수 있다는 전제를 두고
방어적인 코드를 작성하는 것이 중요하다는 점을 배웠습니다.

---

## 2. 데이터 조회 결과 불일치 문제

### 📌 문제 상황

운영자가 조회한 데이터와 실제 DB 데이터가 일치하지 않는 문제가 발생했습니다.

### 🔍 원인

조회 조건이 복잡해지면서 일부 조건이 누락되거나 잘못 적용되어
의도하지 않은 데이터가 조회되는 문제가 있었습니다.

또한, MyBatis 쿼리 작성 시 조건 분기 처리에서 오류가 발생한 부분이 있었습니다.

### 🛠 해결 방법

* 쿼리 조건을 재검토하고 누락된 조건 추가
* MyBatis 동적 쿼리 로직 정리 및 가독성 개선
* 주요 조회 기능에 대해 테스트 케이스 기반 검증 수행

### 📈 결과

* 데이터 조회 정확도 향상
* 운영 중 데이터 불일치 문제 감소

### 💡 배운 점

데이터 조회 기능에서는 쿼리의 정확성이 매우 중요하며,
조건이 복잡할수록 명확한 구조와 검증이 필요하다는 것을 경험했습니다.
