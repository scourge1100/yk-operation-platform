# 프로젝트 개요

본 문서는 **열혈강호 W 운영툴 시스템**의 개발 배경, 목표, 그리고 시스템 구성 개요를 설명한다.

---

## 프로젝트 배경

열혈강호 W 운영툴은 MMORPG **열혈강호 W**의 라이브 서비스 운영을 지원하기 위해 개발된 웹 기반 관리 시스템이다.

게임 서버는 보안 정책에 따라 외부 네트워크에서 직접 접근할 수 없도록 구성되어 있었다.  
따라서 운영자는 공지 등록, 아이템 지급, 유저 데이터 조회와 같은 운영 작업을 서버에 직접 접속하여 처리할 수 없는 환경이었다.

이러한 제약을 해결하기 위해 운영자가 웹 기반 인터페이스를 통해 게임 서비스를 관리할 수 있도록 별도의 운영툴 시스템을 구축하였다.

---

## 문제 정의

게임 출시 준비 단계에서 운영팀이 사용할 수 있는 운영 관리 시스템이 존재하지 않는 상태였다.

게임 서비스 운영을 위해 다음과 같은 기능이 필요했다.

- 게임 내 공지 등록 및 관리
- 특정 유저 또는 캐릭터 대상 아이템 지급
- 이벤트 관리
- 유저 및 캐릭터 데이터 조회
- 운영 작업 로그 관리

이러한 기능이 없는 경우 운영자는 서버에 직접 접근하거나 데이터베이스를 수동으로 조작해야 하며, 이는 보안 위험과 운영 비효율을 초래할 수 있었다.

---

## 프로젝트 목표

본 프로젝트의 주요 목표는 다음과 같다.

- 게임 서비스 운영을 위한 필수 관리 기능 제공
- 운영자가 서버 접속 없이 웹 기반 인터페이스를 통해 운영 작업 수행
- 게임 서버와의 안전한 통신 구조 구현
- 운영 자동화 및 관리 기능을 통한 운영 효율성 향상
- 향후 이벤트 및 운영 기능 확장을 고려한 유연한 메뉴 및 권한 구조 설계

---

## 시스템 구성 개요

운영툴 시스템은 **AWS 기반 3-Tier Architecture** 구조로 설계되었다.

게임 서버는 외부 네트워크에서 직접 접근할 수 없도록 분리되어 있으며, 운영툴에서 발생하는 모든 요청은 **인터페이스 서버를 통해 게임 서버로 전달되는 구조**로 구성되었다.

이 구조를 통해 다음과 같은 효과를 얻을 수 있었다.

- 게임 서버 외부 노출 차단을 통한 보안 강화
- 운영툴과 게임 서버 간 통신 경로 통제
- 운영툴 장애 발생 시 게임 서비스 영향 최소화
